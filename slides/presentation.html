Liora — Data Science Project

Rakuten France
Multimodal Product Classification

Predict the category of 84,916 e-commerce listings from their title, description and image — a 27-class, multilingual, multimodal problem.
Team project — Sandeep Grover · Jonathan Vints · Thomas Maisch
Public benchmark: Rakuten France Multimodal Product Data Classification
ENS Challenge Data (challenge 35) · also on Kaggle · Amoualian et al., ECIR 2021

The business case

The problem
  • Rakuten = global marketplace, millions of sellers
  • Every product needs the right catalogue category
  • Wrong category → invisible in search & recommendations
Why it is hard at scale
  • Free-form titles in many languages
  • A third of products have no description
  • Manual sorting of 85,000+ listings is slow & error-prone
The solution
  • Predict the category automatically
  • From the product's own text and image

Three problems in one glance: multiple languages · HTML inside descriptions · often no description at all

Imbalance · chi-square goodness-of-fit

Imbalance is real
  • Biggest 10,209 vs smallest 764 — a 13.4× gap
  • Chi-square goodness-of-fit: χ² = 36,570, p < 0.001
  • Classes are significantly non-uniform — not chance
  • Potential solution: report weighted-F1 (challenge metric) + macro-F1 for rare classes; class weighting; stratified split
Ref: Amoualian et al. (2021), ECIR — Rakuten dataset

Missing data · MNAR

Missingness is informative
  • Missing rate varies strongly by category
  • Chi-square independence: χ² = 43,758, V = 0.72, p < 0.001
  • So absence depends on the class — this is MNAR, not random (MCAR/MAR)
  • Potential solution: add a desc_missing flag — never drop the rows
Ref: Little & Rubin (2019), Statistical Analysis with Missing Data

Text length · Kruskal-Wallis

Length differs by class
  • Median ~314 chars, but heavily skewed with a long tail
  • Kruskal-Wallis (non-parametric ANOVA): H = 35,757, p < 0.001
  • Length varies significantly across categories (skewed → no normal ANOVA)
  • Potential solution: keep designation_len / description_len
Ref: Kruskal & Wallis (1952), JASA 47(260):583-621

Languages · Wilson 95% CI

Real language strata
  • Mostly French (~81%; ~71% on titles alone), plus English (~10%), German and a European tail
  • Wilson 95% CI stays valid for the rare languages (Wald breaks near 0)
  • Confirms English/German are genuine sub-populations, not noise
  • Potential solution: a French / multilingual language model — CamemBERT (French RoBERTa), FlauBERT or XLM-R
Ref: Wilson (1927), JASA 22(158):209-212

Text preprocessing — clean, then split

Step 1 · per listing, before the split
Clean the text
  • Merge title + description → one field, lowercase (keep French accents)
  • Strip HTML (BeautifulSoup), URLs, e-mails, numbers
  • Tokenize (TweetTokenizer) · remove FR/EN/DE stopwords (nltk) · Snowball French stemming
  • Flag missing descriptions
→
Step 2
Split 80 / 20
  • 80% train · 20% test
  • Stratified — both keep the 27-class balance
  • Test held out — used only to evaluate
Cleaning is per-listing, so it is safe on the whole dataset before the split. TF-IDF (next) learns corpus-wide statistics, so it is fit only after the split, on the training set.
Ref: Richardson, Beautiful Soup (HTML parser, crummy.com); Amoualian et al. (2021), ECIR — Rakuten dataset

Word frequency: most vs least frequent

Frequent ≠ useful
  • Most frequent = stopwords (de, la, et, pour…) → removed by nltk FR/EN/DE stopword list (IDF down-weights any survivors)
  • Least frequent = one-off typos & serial numbers (appear once)
  • Both extremes are dropped: min_df = 3 + the 20,000-feature cap
  • The useful, discriminative words sit in between
Ref: Sparck Jones (1972), J. Documentation 28(1)

Step 3 — TF-IDF: the feature matrix

How each score is built
  • Uses bigrams too (word-pairs): "jeux video"
  • TF: how often a word appears in a listing
  • IDF: how rare it is across all listings
  • Score = TF × IDF: rare words win, filler → ~0
Worked example
  • "nintendo" in ~200 of 85,000 → IDF ≈ 6.1 → high
  • "de" in ~80,000 → IDF ≈ 0.06 → ~zero
Ref: Sparck Jones (1972), J. Documentation 28(1); Pedregosa et al. (2011), scikit-learn
The matrix structure
  • Rows = listings · Columns = 20,000 terms
  • Each cell = a TF-IDF score · ~99% zero (sparse)
  • Fit on train; both train & test transformed to sparse matrices
